# SkillSwap Complete Color Palette Guide
*Primary Blue Focus with Strategic Orange & Green Accents*

## Primary Color Scheme: Blue Primary + Selective Orange & Green Usage

### Background Colors

#### **Main Backgrounds**
- **Website Background**: `gray.50` - Clean, professional base for entire site
- **Card/Component Background**: `white` - Clean contrast for cards, modals, forms
- **Section Backgrounds**: `blue.50` - Very subtle blue tint for hero sections, features
- **Sidebar/Navigation Background**: `white` - Clean background for navigation elements
- **Footer Background**: `gray.100` - Slightly darker than main background

### Primary Brand Colors (Blue-Focused)

#### **Main Brand Colors (Blue)**
- **Primary Blue**: `blue.500` - Main brand color, all primary buttons, navigation, CTAs
- **Secondary Blue**: `blue.600` - Headers, navbar brand, important elements, logo text
- **Light Blue**: `blue.100` - Background highlights, hover states, badge backgrounds
- **Dark Blue**: `blue.700` - Button hover states, active navigation, pressed states

#### **Strategic Accent Colors** *(Use Sparingly)*
- **Orange**: `orange.500` - ONLY for urgent notifications, warnings, or special badges
- **Light Orange**: `orange.100` - Warning backgrounds, alert highlights  
- **Green**: `green.500` - ONLY for success states, completed achievements, confirmations
- **Light Green**: `green.100` - Success backgrounds, completion indicators

### Text Colors

#### **Primary Text**
- **Main Headings**: `gray.800` - H1, H2, primary titles
- **Secondary Headings**: `gray.700` - H3, H4, section titles
- **Body Text**: `gray.600` - Main content, descriptions
- **Muted Text**: `gray.500` - Placeholder text, timestamps, metadata
- **Brand Text**: `blue.600` - Logo text, brand elements, featured text

#### **Interactive Text**
- **Links**: `blue.500` - Clickable text links
- **Link Hover**: `blue.600` - Link hover states
- **Active Links**: `blue.700` - Currently active navigation items

### Component-Specific Colors

#### **Buttons** *(Blue Primary)*
- **Primary Button**: 
  - Background: `blue.500`
  - Hover: `blue.600`
  - Active: `blue.700`
  - Text: `white`

- **Secondary Button**: 
  - Background: `gray.100`
  - Hover: `gray.200`
  - Text: `blue.600`
  - Border: `gray.300`

- **Success Button** *(Limited Use)*: 
  - Background: `green.500`
  - Hover: `green.600`
  - Text: `white`
  - USE ONLY for final completion actions

- **Outline Button**: 
  - Background: `transparent`
  - Border: `blue.500`
  - Text: `blue.500`
  - Hover Background: `blue.50`

#### **Status & Feedback Colors** *(Minimal Color Usage)*
- **Success/Completed**: `green.500` - ONLY for completed skills, final success messages
- **Warning/Alert**: `orange.500` - ONLY for important warnings, urgent notifications
- **Primary/Default**: `blue.500` - All standard actions, navigation, brand elements
- **Error**: `red.500` - Error messages, failed states
- **Info**: `blue.400` - Information messages, tips

#### **Skill Categories** *(Blue-Focused)*
- **All Skill Categories**: `blue.500` - Consistent blue for all skill types
- **Featured Skills**: `blue.600` - Slightly darker blue for emphasis
- **Completed Skills**: Add `green.500` small indicator/checkmark only
- **Popular Skills**: `blue.400` - Lighter blue variation

#### **Navigation & UI Elements**
- **Navbar**: 
  - Background: `white`
  - Border: `gray.200`
  - Brand: `blue.600`
  - Links: `blue.500`
  - Active item: `blue.700`
  - Hover: `blue.50` background

- **Badges & Tags** *(Simplified)*:
  - Default: `blue.100` background, `blue.600` text
  - Important: `blue.500` background, `white` text
  - Success: `green.100` background, `green.600` text (completion only)
  - Warning: `orange.100` background, `orange.600` text (alerts only)

#### **Cards & Containers**
- **Standard Card**: 
  - Background: `white`
  - Border: `gray.200`
  - Shadow: `md`
  - Hover Border: `blue.200`

- **Active/Selected Card**: 
  - Background: `white`
  - Border: `blue.500`
  - Shadow: enhanced

- **Completed Card**: 
  - Background: `white`
  - Border: `gray.200`
  - Small green checkmark indicator only

#### **Forms & Inputs**
- **Input Background**: `white`
- **Input Border**: `gray.300`
- **Input Focus Border**: `blue.500`
- **Input Success State**: `green.500` border (form completion only)
- **Input Error Border**: `red.500`
- **Input Text**: `gray.700`
- **Placeholder Text**: `gray.400`
- **Label Text**: `gray.700`

#### **Progress & Interactive Elements**
- **Progress Bars**:
  - Background: `gray.200`
  - Progress: `blue.500`
  - Complete: Small `green.500` indicator at end

- **Rating Stars**: `blue.400` - Consistent with brand
- **Notification Indicators**: `orange.500` - Only for urgent notifications
- **Online Status**: `green.400` - User availability only

### Simplified Usage Guidelines

#### **When to Use Each Color:**

**Blue (90% of colored elements):**
- All buttons, links, navigation
- Brand elements, headers
- Interactive elements, forms
- Progress indicators, ratings
- Skill categories and cards

**Green (5% - Success Only):**
- Completed skill indicators
- Form submission success
- Achievement notifications
- Final completion states

**Orange (5% - Alerts Only):**
- Urgent notifications
- Important warnings
- System alerts

#### **Page Structure Example**
```
Website Background: gray.50
  └── Header/Navbar: white + blue elements
  └── Main Content: gray.50
      ├── Hero Section: blue.50 + blue.500 buttons
      ├── Content Cards: white + blue accents
      └── Sidebar: white + blue navigation
  └── Footer: gray.100 + blue links
```
Icons :-
Most Common: gray.600 (standard UI icons)
Interactive: blue.500 (clickable icons)
Success: green.500 (completion only)
Warning: orange.500 (alerts only)
Brand: blue.600 (logo, brand elements)
### Best Practices - Blue-First Approach

1. **Default to Blue**: When in doubt, use blue variants
2. **Minimize Color Diversity**: Keep it simple and consistent
3. **Green = Success Only**: Don't use green for general elements
4. **Orange = Urgent Only**: Reserve orange for important alerts
5. **Gray for Content**: Use gray scale for text hierarchy
6. **White for Clean Spaces**: Cards, forms, navigation backgrounds
7. **Consistent Interactions**: All clickable elements use blue
8. **Brand Recognition**: Blue creates strong brand consistency
